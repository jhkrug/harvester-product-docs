= Introduction to Backup and Restore
:revdate: 2026-07-15
:page-revdate: {revdate}

A robust backup and restore strategy is essential for protecting the workloads that run on {harvester-product-name-tm}. Because {harvester-product-name} manages virtual machines as native Kubernetes objects (through KubeVirt), a complete strategy must protect both the _data_ stored in virtual machine volumes and the _Kubernetes resources_ that define each virtual machine and its dependencies.

This section introduces backup and restore for {harvester-product-name}, defines the key terms, helps you set recovery objectives, and describes the available tools and how to integrate them.

== What to protect

When you plan how to protect a virtual machine, consider the following components:

* *Virtual machine volumes*: The root volume and any data volumes that hold the guest operating system and application data. These volumes may be provisioned by the built-in {longhorn-product-name-tm} storage or by an external xref:storage/csidriver.adoc[Container Storage Interface (CSI) driver].
* *Virtual machine definitions*: The Kubernetes resources that describe the virtual machine, such as the `VirtualMachine` and `VirtualMachineInstance` objects, along with dependencies like persistent volume claims, secrets (for example, `cloud-init` configuration), and network attachment definitions.
* *Cluster configuration*: The broader cluster state that is required to rebuild an environment during disaster recovery.

== Getting started

If you are new to backup and restore on {harvester-product-name}, the simplest option is the built-in virtual machine backup, snapshot, and restore functionality, which requires no additional software and exports backups to an NFS or S3-compatible target. See xref:backup-restore/integration-built-in.adoc[Built-in Virtual Machine Backup, Snapshot & Restore].

[NOTE]
====
Built-in backup support is currently limited to {longhorn-product-name} volumes. To back up virtual machines that use volumes provisioned by an external CSI driver, use one of the Kubernetes-native tools described in xref:backup-restore/tools.adoc[Backup Tools].
====
